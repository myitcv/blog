Computers are dumb

At the most basic

  • Inputs (hardware/data): mouse, keyboard, camera, files...
  • Processing (software): programs, code
  • Output (hardware/data): monitor, printer, web page...

What is hardware?

  • Hard drive - long term storage
  • RAM - short term memory (needs power)
  • CPU/processor - follows instructions (maths and logic)
  • (obviously an incomplete list)

What is software?

  • A series of instructions
  • A recipe if you will
  • Needs to be stored somewhere

What is data?

  • Another input/output
  • Needs to be stored somewhere
  • Classically understood as "files"

We understand computers in terms of stacks

  • Users (you and me)
  • Applications (Word, Excel)
  • Operating Systems (Windows, Mac OS X, Linux)
  • Firmware (a special kind of software)
  • Hardware (Samsung, MacBook Pro)

What does this have to do with our web application?

Picking up the Twitter example

  • Twitter is an application
  • A very big application
  • ...that scales to millions of users
  • ...all over the world
  • But an application nonetheless

But I can't install Twitter...

  • We distinguish between clients and servers
  • Server: the software + hardware that "provides" Twitter

  • Client: the software + hardware that "uses" Twitter

    • Mobile phone (iPhone, Android)
    • Personal computer
    • Another server (using the Twitter data API)
  • Coffee shop analogy

But "where" is Twitter?

  • Computers connect to networks
  • Computers on networks have IP addresses: unique identifier
  • e.g. 163.119.33.119
  • eduroam is the network here at LBS
  • eduroam is connected to the internet
  • The Twitter hardware is similarly networked...
  • ...and ultimately connected to the internet
  • But I typed http://twitter.com/#lbs?

A quick primer on internet addresses

  • http://twitter.com/#lbs is a URL
  • twitter.com is a domain
  • Domain Name System (DNS) translates domain names to an IP address
  • This locates the Twitter hardware on the internet
  • Twitter servers know how to "answer" or serve http://twitter.com/#lbs

Terminology refresher

  • Program = code = application (slight abuse of termology here...)
  • Programs are stored, like data, in files on disk (hard drive?)
  • When we "run" a program it is loaded into memory (RAM)
  • Processor then steps through instructions (and may repeat)
  • A running program is called a process, e.g. Chrome

File system (storage)

  • / - root, everything sits under here
  • /home - user directories
  • /home/lbs - the directory for your files, your home directory
  • /home/lbs/my_file.txt - a text file in the lbs home directory

File system (more)

  • /etc - operating system settings
  • /bin - essential bin-aries (programs)
  • /boot - files essential to boot (start) the operating system
  • /tmp - temporary area, every file "below" here gets deleted when the machine restarts

Let's have a drive around

  • Terminal = command line = command prompt
  • Browser
  • Text Editor

Terminal commands

  • ls - list directory contents
  • cd - change current directory
  • pwd - print the current directory
  • mkdir - make/create a directory
  • rm - remove a file/directory
  • touch - create an empty file
  • cp - copy a file/directory
  • man - show a manual for a command

Examples

  • ls /home/lbs - show the contents of our home directory
  • pwd - print the current directory
  • cd /tmp - change the current directory to /tmp
  • pwd - print the current directory
  • mkdir blah - create a directory called blah in the current directory
  • cd blah - change the current directory to blah
  • touch test - create an empty file called test in the current directory
  • ls . - show the contents of the current directory
  • cd .. - go up a directory
  • rm -r blah - remove the directory we just created
  • man rm - show the manual for the rm command

OK, lots of new stuff here...

  • Windows calls directories "folders"
  • Within the terminal, a program called bash is accepting our "input"
  • Everything is case sensitive
  • Difference between files and directories
  • Filenames containing spaces create real problems...use underscore

What on earth in Cloud Computing?

Wikipedia: use of computing resources (hardware and software) that are delivered as a service over a network (typically the Internet)

Covers compute, network and storage services

Why Cloud Computing?

  • Fast
  • Easily scalable
  • Spans geographies

Here comes another stack....

  • Software as a Service (SaaS) - Gmail, Salesforce
  • Platform as a Service (PaaS) - OpenShift, Heroku
  • Infrastructure as a Service (IaaS) - Amazon Web Services

More on PaaS

  • Runs on IaaS
  • i.e. runs within an operating system (typically a Linux distribution)
  • PaaS is itself a set of programs that host applications
  • Take away management headaches of running those applications
  • Easy deployment

OpenShift

  • An open-source PaaS
  • Provided by RedHat, who also provide a distribution of Linux
